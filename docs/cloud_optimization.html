v4.3 클라우드 환경 최적화

Streamlit Cloud의 메모리·타임아웃 제한을 극복하기 위한 3가지 핵심 개선

Before — 문제점

  • 위젯 클릭 시 전체 데이터 수집 재실행 (2~3분)
  • 건축물대장 지오코딩 순차 처리 (~80초)
  • Streamlit Cloud 응답 타임아웃 → 홈 화면 복귀
  • osmnx 1.9.4 (존재하지 않는 버전) 빌드 실패
→

After — 해결

  • @st.cache_data 캐싱 → 재실행 시 즉시 반환
  • ThreadPoolExecutor 병렬 지오코딩 (~15초)
  • 첫 분석 이후 타임아웃 없음
  • osmnx>=2.0.0 하한 지정 → 빌드 안정
🗄️
IMPROVEMENT 01

@st.cache_data 캐싱

_cached_collect_all()과 _cached_income_rent()로 API 수집 결과를 GeoJSON 직렬화하여 1시간 동안 캐싱합니다.

같은 지역·업종 조합이면 API를 다시 호출하지 않고 캐시에서 즉시 반환합니다.

2~3분 → 0 초 (캐시 히트 시)
⚡
IMPROVEMENT 02

지오코딩 병렬화

건축물대장 API에서 수집한 주소 ~200건을 카카오 지오코딩 API로 좌표 변환할 때, ThreadPoolExecutor(10)으로 10개씩 동시 처리합니다.

~80초 → ~15 초 (5배 단축)
📦
IMPROVEMENT 03

의존성 호환

고정 버전(==)에서 하한 지정(>=)으로 변경하여 Streamlit Cloud 빌드 환경과의 호환성을 확보합니다.

osmnx==1.9.4 (미존재) → osmnx>=2.0.0

빌드 실패 → 안정 빌드
